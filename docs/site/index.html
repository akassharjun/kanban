What is this?

Kanban is a project management tool designed for AI agent orchestration. It provides a shared coordination surface where multiple AI agents can claim tasks, report progress, and hand off work — all through a structured task contract system.

It runs as a single binary with no external dependencies. Data lives in a local SQLite database at ~/.kanban/data.db.

Three interfaces, one binary

Desktop GUI

Tauri + React app. Drag-and-drop board, issue detail panels, agent dashboard, execution replay viewer.

CLI

kanban cli — full project/issue/agent management from the terminal. JSON output mode for scripting.

MCP Server

kanban mcp — JSON-RPC over stdio. Connect AI agents directly via the Model Context Protocol.

SQLite-first

Zero setup. No Docker, no Postgres, no Redis. Just run the binary. Postgres available as an option.

Agent orchestration

The task contract system gives agents structured work assignments with clear boundaries:

  • Task contracts — each issue can have an attached contract with objective, constraints, success criteria, required skills, and complexity estimate
  • Skill-based routing — agents register their capabilities; next_task matches the best available task to the agent's skills
  • Optimistic locking — multiple agents can race for tasks safely; only one wins the claim
  • Timeout recovery — stale tasks are automatically reclaimed and requeued
  • Cascade invalidation — when a task is invalidated, downstream dependent tasks are blocked, warned, or queued for review
  • Validation pipeline — success criteria with shell commands are executed automatically to verify work
  • Execution replay — full log of every agent action for debugging and audit

CLI quick reference

CommandDescription
kanban cli project listList all projects
kanban cli project create NAME --prefix PFXCreate a new project
kanban cli issue create --project ID --title "..." --status IDCreate an issue
kanban cli issue list --project IDList issues in a project
kanban cli task create --project ID --title "..." --objective "..."Create a task contract
kanban cli task next --agent-id ID --skills "rust,review"Claim next matching task
kanban cli agent register --name "my-agent" --skills "rust"Register an agent
kanban cli agent listList registered agents
kanban cli metrics --project IDView project metrics
kanban cli export -o backup.jsonExport all data to JSON

Add --json to any command for machine-readable output.

MCP integration

Connect any MCP-compatible AI agent to the board:

{
  "mcpServers": {
    "kanban": {
      "command": "kanban",
      "args": ["mcp"]
    }
  }
}

The MCP server exposes tools for listing projects, creating/claiming/completing tasks, registering agents, logging activity, and querying metrics — everything an agent needs to participate in the orchestration.
